# 🧪 QA Portfolio: Health Tracking Application

## 📌 Project Overview
This repository contains my QA documentation and API testing artifacts for **Healthy Hub** – a web and mobile application for tracking daily calories, water intake, and personal health goals.

## 🎯 What I Did
* **API Testing:** Conducted deep API testing using **Postman** and **Swagger (OAS 3.1)**. 
* **Defect Localization:** Used **Chrome DevTools (Network/Console)** to investigate UI freezes and identify root causes, differentiating between frontend JS exceptions and backend errors.
* **Bug Reporting:** Documented over 15 bugs, including critical logic errors, missing data validations, and contract violations between Swagger documentation and backend behavior.

## 📂 Repository Contents
* `1_Bug_Reports/`: Detailed bug reports formatted in Markdown, highlighting boundary value issues and API routing errors.
* `2_Test_Documentation/`: Test cases and checklists.
* `3_API_Testing_Postman/`: Exported Postman collection containing HTTP methods (GET, POST, PUT, PATCH) and environment variables used for the testing process.

## 🛠 Tools Used
* **API:** Postman, Swagger, REST principles
* **Debugging:** Chrome DevTools
* **Formats:** JSON, Markdown
