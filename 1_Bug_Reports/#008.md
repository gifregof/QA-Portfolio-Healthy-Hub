# [Blocker] 400 Bad Request in POST /user/food-intake-v2 due to Schema Mismatch (Object vs Array)

## Metadata
- **ID:** #8
- **Reporter:** Vyskrebstsov O.
- **Severity:** Blocker
- **Priority:** High
- **Type:** Functional
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The `POST /user/food-intake-v2` endpoint rejects valid JSON (according to Swagger documentation) with a `400 Bad Request` error because it expects arrays for nutritional fields instead of objects.

## Description
There is a critical discrepancy between the OAS 3.1 documentation and the backend implementation. The Swagger "Example Value" and "Schema" represent "breakfast", "lunch", "dinner", and "snack" as objects `{}`. However, the server returns a validation error stating these fields must be arrays `[]`.

## Pre-conditions
1. Open the API documentation: `https://healthy-hub-qa.b.goit.study/api-docs/#/`
2. Ensure the user is authorized with a valid Bearer token.

## Steps to Reproduce
1. Locate the `POST /user/food-intake-v2` endpoint.
2. Click the **"Try it out"** button.
3. Insert the exact JSON from the documentation's "Example Value" into the request body:
   ```json
   {
     "date": "2023-09-24T19:35:04.610Z",
     "breakfast": { "calories": 0, "carbohydrates": 0, "protein": 0, "fat": 0 },
     "lunch": { "calories": 0, "carbohydrates": 0, "protein": 0, "fat": 0 },
     "dinner": { "calories": 0, "carbohydrates": 0, "protein": 0, "fat": 0 },
     "snack": { "calories": 0, "carbohydrates": 0, "protein": 0, "fat": 0 },
     "total": {
       "water": { "goal": 1500, "used": "100" },
       "calories": { "goal": 2733, "used": 0 },
       "carbohydrates": { "goal": 170, "used": 0 },
       "protein": { "goal": 127.5, "used": 0 },
       "fat": { "goal": 56, "used": 0 }
     }
   }
4. Click the "Execute" button.

## Actual Result
- **Status Code:** 400 Bad Request
- **Response Body:** `{"message": "Breakfast should be an array"}` (and similar for other fields).

## Expected Result
- **Status Code:** 200 OK or 201 Created.
- **Response Body:** `{"message": "Successful operation"}`
- **Behavior:** The API should accept the request since it matches the official documentation, or the documentation should be updated to reflect that these fields must be arrays.
