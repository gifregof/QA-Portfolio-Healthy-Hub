# [Blocker] Infinite loading in Swagger UI for GET /user/recommended-food when unauthorized

## Metadata
- **ID:** #14
- **Reporter:** Vyskrebstsov O.
- **Severity:** Blocker
- **Priority:** High
- **Type:** Functional / UI
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The Swagger UI hangs in an infinite loading state instead of displaying a `401 Unauthorized` response when executing the `GET /user/recommended-food` request without an authorization token.

## Description
When a user attempts to call the protected endpoint `GET /user/recommended-food` without providing a Bearer token, the Swagger UI interface fails to process the response. Instead of rendering the status code and error message, the "Execute" button trigger results in a continuous loading animation.

## Pre-conditions
1. Open the API documentation: `https://healthy-hub-qa.b.goit.study/api-docs/#/`
2. Ensure no authorization token is set (Authorize field is empty).

## Steps to Reproduce
1. Locate the `GET /user/recommended-food` endpoint.
2. Click the **"Try it out"** button.
3. Click the **"Execute"** button without providing any parameters or tokens.
4. Observe the UI response.

## Actual Result
- The request does not complete.
- Swagger UI shows a permanent **"Loading..."** status.
- No response body or status code is displayed.

## Expected Result
- **Status Code:** 401 Unauthorized.
- **Behavior:** Swagger UI should immediately display the error response received from the server.

## Additional Info
- This issue is specific to the **Swagger UI** implementation. Testing the same scenario via **Postman** correctly returns a `401 Unauthorized` status.
- The same "infinite loading" bug is reproducible on the following endpoints:
    - `GET /user/graph`
    - `GET /user/graph-v2`
- This is a **Blocker** because it breaks the usability of the documentation for unauthorized testing scenarios.
