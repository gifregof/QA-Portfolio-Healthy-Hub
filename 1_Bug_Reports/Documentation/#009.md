# [Bug] Inconsistent data structure in POST /user/food-intake-v2 response (Array to Object conversion)

## Metadata
- **ID:** #9
- **Reporter:** Vyskrebstsov O.
- **Severity:** Major
- **Priority:** High
- **Type:** Functional
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The `POST /user/food-intake-v2` endpoint changes the data structure in the response body without documentation, returning an object instead of the array that was sent and expected.

## Description
When sending a valid array of items to the `breakfast` field (as required by the backend in Bug #008), the API processes the request but returns the data in a different, undocumented format. This inconsistent behavior makes it impossible for the client-side to predict and parse the response correctly.

## Pre-conditions
1. Open the API documentation: <https://healthy-hub-qa.b.goit.study/api-docs/#/>
2. Ensure the user is authorized with a valid Bearer token.

## Steps to Reproduce
1. Locate the `POST /user/food-intake-v2` endpoint.
2. Click the **"Try it out"** button.
3. Use the following JSON in the request body (using an array as the backend currently requires):
   ```json
   {
     "breakfast": [
       {
         "name": "Test",
         "calories": 1,
         "carbohydrates": 1,
         "protein": 1,
         "fat": 1
       }
     ]
   }
   ```
4. Click the "Execute" button.
5. Examine the structure of the returned data in the "Response body".

## Actual Result
- The data structure in the response is modified (e.g., converted from an array to an object or a different nested format) without any explanation or documentation in Swagger.

## Expected Result
- The response structure should remain consistent with the input or strictly follow the schema defined in the documentation.
- If the transformation is intentional, it must be explicitly documented in OAS 3.1.
