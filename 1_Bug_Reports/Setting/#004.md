# [Critical] PATCH /auth/settings requires all mandatory fields (behaves like PUT)

## Metadata
- **ID:** #4
- **Reporter:** Vyskrebstsov O.
- **Severity:** Critical
- **Priority:** High
- **Type:** Functional
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The `PATCH /auth/settings` endpoint incorrectly requires all mandatory fields to be present in the request body, behaving like a `PUT` method instead of allowing partial updates.

## Description
According to REST standards and the provided Swagger documentation, the `PATCH` method should allow updating individual fields. However, the API returns a validation error if any mandatory field (like "gender") is missing from the request body, forcing the client to send the entire object.

## Pre-conditions
1. Open Swagger UI: <https://healthy-hub-qa.b.goit.study/api-docs/#/>
2. Ensure the user is authorized with a valid Bearer token.

## Steps to Reproduce
1. Locate the [PATCH /auth/settings](https://healthy-hub-qa.b.goit.study/api-docs/#/Auth/patch_auth_settings) endpoint.
2. Click the **"Try it out"** button.
3. Modify the request body to include only one field, for example:
   ```json
   {
     "name": "Dave"
   }
   ```
4. Click the "Execute" button.

## Actual Result
- **Status Code:** 400 Bad Request
- **Response Body:** `{"message": "Gender is required"}` (or similar validation errors for other missing mandatory fields).

## Expected Result
- **Status Code:** 200 OK (or 201 as per current API behavior)
- **Response Body:** `{"message": "Successful operation"}`
- **Behavior:** The system should update only the "name" field and retain existing values for other fields.

## Additional Info
The API implementation violates the PATCH method contract. While Swagger lists the fields, it does not specify that they are all mandatory for a partial update request.
