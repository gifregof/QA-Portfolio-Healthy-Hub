# Bug Report: Server Crash (500 Internal Server Error) on Avatar Upload
## Metadata
- **ID:** #17
- **Reporter:** Vyskrebstsov O.
- **Severity:** High
- **Priority:** Medium
- **Type:** Functional

## Summary
The server returns a `500 Internal Server Error` and exposes a Node.js stack trace when an avatar is uploaded using an incorrect field name (key) in a `multipart/form-data` request.

## Environment
* **URL:** <https://healthy-hub-qa.b.goit.study/api-docs/#/Auth/patch_auth_avatar>
* **Method:** `PATCH`
* **Auth:** Bearer Token required
* **Tool:** Postman Desktop App — Reproducible (Result: 500 Internal Server Error)
* **Tool:** Postman Web Version — Not reproducible (Result: 400 Bad Request)

## Steps to Reproduce
1. Open Postman and create a new `PATCH` request.
2. Enter the endpoint URL: <https://healthy-hub-qa.b.goit.study/api-docs/#/Auth/patch_auth_avatar>.
3. In the **Authorization** tab, provide a valid Bearer Token.
4. In the **Body** tab, select `form-data`.
5. Add a new key named `avatar` (or any name other than `file`).
6. Change the key type from `Text` to `File`.
7. Select a valid image file (PNG/JPG).
8. Send the request.

## Actual Result
The server crashes and returns a `500 Internal Server Error` along with a detailed stack trace in HTML format.

## Expected Result
The server should return a `400 Bad Request` status code with a descriptive error message (e.g., `"Invalid field name"` or `"File is required"`), indicating that the expected key is `file`.

## Technical Analysis:
The crash is caused by an unhandled exception in the global error handler (app/app.js:30). The server attempts to pass a string constant LIMIT_UNEXPECTED_FILE (from Multer) into the Express res.status() or res.send() method, which expects a valid integer HTTP status code. This results in a RangeError [ERR_HTTP_INVALID_STATUS_CODE] and a complete server-side failure.
