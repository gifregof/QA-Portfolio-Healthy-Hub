# [Blocker] Missing Frontend Validation Leads to UI Freeze (JavaScript Exception)

## Metadata
- **ID:** #14
- **Reporter:** Vyskrebstsov O.
- **Severity:** Blocker
- **Priority:** High
- **Type:** Functional / UI
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

**Summary:** Infinite loading spinner (UI freeze) when attempting to send a `GET /user/recommended-food` request with an empty required `count` parameter.

**Steps to Reproduce:**
1. Open the [Swagger UI](https://healthy-hub-qa.b.goit.study/api-docs/#/) page.
2. Expand the `GET /user/recommended-food` endpoint.
3. Leave the `count` parameter input field empty.
4. Click the "Execute" button.

**Actual Result:**
- The UI freezes in an infinite loading state (spinner).
- The request is not dispatched to the server (no outgoing request in the DevTools **Network** tab). A JavaScript exception is thrown in the DevTools **Console**: `Error: Required parameter count is not provided`. No validation error message is displayed to the user on the UI.

**Expected Result:**
- **Status Code:** 401 Unauthorized.
- **Behavior:** Swagger UI should immediately display the error response received from the server.
  
**Root Cause Analysis (RCA):**
- The issue is localized to the frontend UI.
- The Swagger JavaScript application crashes when trying to build the request URL without the required parameter, preventing the execution of further code (including the command to stop the loading spinner).
