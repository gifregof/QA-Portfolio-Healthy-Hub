# [Critical] Swagger UI ignores "count" parameter in GET /user/recommended-food due to incorrect definition (Path vs Query)

## Metadata
- **ID:** #26
- **Reporter:** Vyskrebstsov O.
- **Severity:** Critical
- **Priority:** High
- **Type:** Functional / Documentation
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The Swagger UI fails to include the "count" parameter in the request URL for `GET /user/recommended-food`, resulting in a `400 Bad Request`. This happens because the parameter is incorrectly defined as a **Path** parameter instead of a **Query** parameter in the documentation.

## Description
When using Swagger UI to test the `GET /user/recommended-food` endpoint, the value entered in the "Count of food" field is ignored. Network DevTools shows that the request is sent without any parameters. Investigation reveals that the OAS 3.1 schema incorrectly defines "count" as part of the path, while the backend expects it as a query string.

## Pre-conditions
1. Open the API documentation: `https://healthy-hub-qa.b.goit.study/api-docs/#/`
2. Ensure the user is authorized with a valid Bearer token.

## Steps to Reproduce
1. Locate the `GET /user/recommended-food` endpoint.
2. Click the **"Try it out"** button.
3. Enter a valid value in the "Count of food" field (e.g., `"5"`).
4. Click the **"Execute"** button.
5. Observe the Request URL in Swagger and the Network tab in DevTools.

## Actual Result
- **Request URL:** `.../api/user/recommended-food` (parameter is missing).
- **Status Code:** 400 Bad Request.
- **Response Body:** `{"message": "Count not specified or incorrect"}`.

## Expected Result
- **Request URL:** `.../api/user/recommended-food?count=5` (parameter should be appended as a query).
- **Status Code:** 200 OK.
- **Behavior:** The API should return a list of recommended foods based on the specified count.

## Additional Info
- Manual testing via **Postman** with the query parameter `?count=5` works correctly and returns `200 OK`.
- The same "Path vs Query" definition error is present in the following endpoints:
    - `GET /user/graph`
    - `GET /user/graph-v2`
- **Action Required:** Update Swagger/OAS annotations to define these parameters as `in: query` instead of `in: path`.
