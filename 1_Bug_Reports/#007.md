# [Bug] Missing numerical range and data type validation in POST /user/food-intake

## Metadata
- **ID:** #7
- **Reporter:** Vyskrebstsov O.
- **Severity:** Major
- **Priority:** High
- **Type:** Functional
- **Environment:** Windows 10 Pro, Google Chrome 146.0.7680.80, API v1.0.0, OAS 3.1

---

## Summary
The `POST /user/food-intake` endpoint lacks proper validation for the "name", "calories", "carbohydrates", "protein", and "fat" fields, allowing invalid data types and extreme numerical values to be processed.

## Description
The API accepts and processes invalid values (e.g., numeric strings for names and extremely large integers for nutritional values) without returning an error. This results in corrupted data being stored and returned in the response.

## Pre-conditions
1. Open the API documentation: `https://healthy-hub-qa.b.goit.study/api-docs/#/`
2. Ensure the user is authorized with a valid Bearer token.

## Steps to Reproduce
1. Locate the `POST /user/food-intake` endpoint.
2. Click the **"Try it out"** button.
3. In the request body, insert the following JSON with invalid values:
   ```json
   {
     "lunch": [
       {
         "name": "3456",
         "calories": 555555555555,
         "carbohydrates": 555555555555,
         "protein": 5555555555555555,
         "fat": 555555555555555
       }
     ]
   }
4. Click the "Execute" button.

## Actual Result
- **Status Code:** 200 OK
- **Response Body:** The server processes the request and returns the same invalid/extreme values in the response body.
